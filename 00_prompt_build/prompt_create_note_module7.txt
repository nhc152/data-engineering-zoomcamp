4. DATA FLOW THỰC TẾ
═══════════════════════════════
- Flow đầy đủ từng pipeline: data đi từ đâu → qua broker/topic nào → qua processor nào → sink đích nào
- Topic lineage: producer nào ghi topic nào, consumer/job nào đọc topic nào
- File nào trigger/chạy file nào
- Thứ tự chạy đúng: docker compose → producer → streaming job → consumer/sink
- Dependency giữa các services: Kafka/Redpanda, Schema Registry nếu có, Flink, Spark, Postgres
- Phân biệt rõ:
  - workshop flow
  - workshop/live flow
  - extras/python JSON flow
  - extras/python Avro flow
  - extras/python Redpanda flow
  - extras/pyflink flow
  - theory/java Kafka Streams flow

═══════════════════════════════
5. ĐIỂM YẾU & THIẾU SÓT
═══════════════════════════════
- Security issues trong code/config thực tế: hardcode broker URL, credentials, secrets, plaintext config
- Delivery semantics gaps: at-most-once / at-least-once / exactly-once, idempotent producer, transactions
- Offset management issues
- Schema evolution / compatibility issues
- Checkpointing/state recovery gaps trong Flink/Spark
- Backpressure, retries, DLQ, poison message handling
- Data quality and validation gaps
- Observability gaps: metrics, logs, tracing, lag monitoring, alerting
- Production readiness gaps: scaling, partitioning, replication factor, retention, compaction
- Testing gaps: unit tests, integration tests, Testcontainers, topology tests
- Những gì repo dạy nhưng không đủ cho Senior DE
