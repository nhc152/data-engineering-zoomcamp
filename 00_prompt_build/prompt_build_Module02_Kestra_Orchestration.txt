## SECTION 5 — Flows 04–05: ETL Postgres Taxi

5.1 Kiến trúc ETL Postgres (diagram):
    [GitHub .csv.gz]
    → wget/gunzip → [Kestra Storage]
    → CREATE TABLE IF NOT EXISTS (main + staging)
    → TRUNCATE staging
    → COPY IN to staging
    → UPDATE (md5 unique_row_id + filename)
    → MERGE staging → main (ON unique_row_id)

5.2 Idempotency Pattern — giải thích tại sao staging + MERGE:
    - md5() hash từ fields: VendorID, pickup_datetime, dropoff_datetime,
      PULocationID, DOLocationID, fare_amount, trip_distance
    - WHEN NOT MATCHED THEN INSERT → chạy lại không duplicate
    - So sánh: INSERT trực tiếp (bad) vs MERGE (idempotent)

5.3 Flow 04 — Manual ETL (giải thích từng task group):
    Task 1: set_label → tag execution
    Task 2: extract → wget + gunzip
    Task 3: if_yellow_taxi → 6 sub-tasks
    Task 4: if_green_taxi → 6 sub-tasks
    Task 5: purge_files → dọn storage
    Schema diff yellow vs green:
      Yellow: tpep_pickup_datetime, tpep_dropoff_datetime
      Green:  lpep_pickup_datetime, lpep_dropoff_datetime, ehail_fee, trip_type
    Full YAML: 04_postgres_taxi.yaml (lấy từ source)
    Test:
      □ Execute yellow/2019/01 → public.yellow_tripdata tồn tại, rows > 0
      □ Execute lại cùng input → row count KHÔNG thay đổi

5.4 Flow 05 — Scheduled ETL + Backfill:
    Khác Flow 04: không inputs, dùng trigger.date
    Date formatting: trigger.date | date('yyyy-MM'), date('yyyy'), date('MM')
    Backfill: start: "2019-01-01T09:00:00Z"
    Full YAML: 05_postgres_taxi_scheduled.yaml (lấy từ source)
    Hướng dẫn kích hoạt backfill từ Kestra UI (step by step)

## SECTION 7 — Flows 08–09: ELT BigQuery

7.1 So sánh ETL vs ELT (bảng):

    | Tiêu chí        | ETL Postgres (F04)        | ELT BigQuery (F08)         |
    |-----------------|---------------------------|----------------------------|
    | Transform ở đâu | Kestra + Postgres SQL      | BigQuery SQL               |
    | Staging         | Staging table trong DB     | External Table → Temp Table|
    | Storage         | Postgres volume            | GCS + BigQuery             |
    | Scale           | Medium (local)             | Large (cloud, petabyte)    |
    | Cost            | Free (local)               | Pay per query              |
    | Idempotency     | MERGE                      | MERGE                      |
    | Partitioning    | Không                      | DATE(tpep_pickup_datetime) |

7.2 ELT Pipeline Architecture (diagram):
    [GitHub .csv.gz]
    → wget/gunzip → [Kestra Storage]
    → Upload → [GCS Bucket]
    → CREATE EXTERNAL TABLE (đọc thẳng từ GCS, không copy data)
    → CREATE TABLE AS SELECT (materialize + add unique_row_id)
    → CREATE main table IF NOT EXISTS (PARTITIONED BY DATE)
    → MERGE into main table

7.3 External Table concept:
    - BQ đọc thẳng từ GCS, không lưu data vào BQ storage
    - Dùng để ingest nhanh → materialize sang native table

7.4 Partitioning trong BigQuery:
    - PARTITION BY DATE(tpep_pickup_datetime)
    - Lợi ích: scan ít data hơn khi filter theo ngày → tiết kiệm cost

7.5 Flow 08 — Manual ELT GCP (giải thích từng task):
    Task 1: set_label
    Task 2: extract → wget + gunzip
    Task 3: upload_to_gcs
    Task 4: if_yellow_taxi → 4 BQ tasks (ext → tmp → main → merge)
    Task 5: if_green_taxi → 4 BQ tasks
    Task 6: purge_files
    Full YAML: 08_gcp_taxi.yaml (lấy từ source)
    Test:
      □ Execute yellow/2019/01 → GCS object exist
      □ BQ table yellow_tripdata có data
      □ Execute lại → row count KHÔNG thay đổi

7.6 Flow 09 — Scheduled ELT GCP + Backfill:
    Tương tự Flow 05 nhưng target GCS + BigQuery
    Full YAML: 09_gcp_taxi_scheduled.yaml (lấy từ source)
    Backfill: 2019-01 → 2020-12
