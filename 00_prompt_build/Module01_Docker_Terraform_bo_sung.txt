BỔ SUNG 3 — Trong S3, phần "Cố tình phá"
Tìm kịch bản cuối cùng trong phần "Cố tình phá" (kịch bản thứ 5).

  → Fix: luôn dùng tên container (--name) làm hostname khi giao tiếp container-to-container

BỔ SUNG 4 — Trong S4, câu 6 SECURITY
Tìm phần trả lời câu hỏi Security (câu 6) trong S4.
Chèn thêm vào cuối phần đó:

- Container đang chạy với root user theo mặc định — rủi ro gì?
  → Nếu có lỗ hổng trong app, attacker có full root access vào container
  → Fix: thêm vào Dockerfile:
    RUN addgroup --system appgroup && adduser --system --ingroup appgroup appuser
    USER appuser

- Service Account Key JSON trên máy cá nhân — rủi ro gì và alternative tốt hơn?

  → Nguyên tắc: credentials không nên tồn tại dưới dạng file trên disk
