═══════════════════════════════
1. CẤU TRÚC THƯ MỤC
═══════════════════════════════
Tree đầy đủ + mục đích từng folder và file

═══════════════════════════════
2. NỘI DUNG CÁC FILE .MD
═══════════════════════════════
Tóm tắt từng file .md theo format:
[Tên file]
- Dạy về: [topic]
- Concepts chính: [list]
- Commands được dùng: [list commands thực tế]
- Lưu ý đặc biệt: [nếu có]

═══════════════════════════════
3. PHÂN TÍCH CODE FILES
═══════════════════════════════
Với mỗi file code, ghi rõ:

[dbt_project.yml]
- Project name, version, profile
- Model configs: materialization mặc định theo layer
- Vars được định nghĩa
- Copy nguyên văn toàn bộ file

[packages.yml / package-lock.yml]
- Packages được dùng, version
- Copy nguyên văn toàn bộ file

[sources.yml]
- Sources được khai báo: database, schema, tables
- Freshness config nếu có
- Copy nguyên văn toàn bộ file

[schema.yml - từng layer]
- Models được document: tên, description
- Tests được định nghĩa: loại test, column target
- Copy nguyên văn toàn bộ file (từng file riêng)

[macros_properties.yml]
- Macros được khai báo, arguments
- Copy nguyên văn toàn bộ file

[seeds_properties.yml]
- Seeds được khai báo, column types
- Copy nguyên văn toàn bộ file

[Từng file .sql trong macros/]
- Macro name, arguments nhận vào
- Logic chính
- Copy nguyên văn toàn bộ file

[Từng file .sql trong models/staging/]
- Source được ref đến
- Transformations thực hiện
- Materialization
- Copy nguyên văn toàn bộ file

[Từng file .sql trong models/intermediate/]
- Models được ref đến (staging hay intermediate khác)
- Transformations thực hiện
- Materialization
- Copy nguyên văn toàn bộ file

[Từng file .sql trong models/marts/]
- Models được ref đến
- Business logic chính
- Materialization
- Copy nguyên văn toàn bộ file

[Seeds .csv]
- Columns và sample data
- Mục đích dùng trong pipeline

═══════════════════════════════
4. DATA FLOW THỰC TẾ
═══════════════════════════════
- DAG đầy đủ: source → staging → intermediate → marts → reporting
- File nào ref đến file nào (lineage graph dạng text)
- Thứ tự chạy đúng theo dependency
- Materialization strategy theo từng layer (view/table/incremental)

5. ĐIỂM YẾU & THIẾU SÓT

- Hardcode issues (values, filter dates, project IDs...)
- Incremental model issues nếu có
- Macro design issues
- Seeds dùng trong production: rủi ro gì
- Documentation gaps
- Những gì repo dạy nhưng không đủ cho Senior DE/Analytics Engineer
