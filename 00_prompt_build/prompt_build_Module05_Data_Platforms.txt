6 câu hỏi, mỗi câu có answer chi tiết + code example:

1. FAILURE: Pipeline chạy 3 tiếng, fail ở transform asset lúc 2h sáng
   - Log xem ở đâu? (Bruin CLI logs, Bruin Cloud UI logs)
   - Asset đang ở trạng thái gì sau khi fail? (partial write, dirty state)
   - bruin run --asset retry hoạt động thế nào — khi nào đủ, khi nào không
   - Fix thực tế: rerun từ failed asset hay rerun toàn bộ pipeline?
   - Bao giờ nên --full-refresh incremental asset và tác động downstream

2. SCALE: Từ demo data (vài MB) lên production data (TB/ngày)
   - Ingestr asset hiện tại có đủ không? Bottleneck ở đâu?
   - Incremental ingestion: làm sao chỉ pull data mới thay vì full reload
   - Date range variables: tại sao start_date/end_date là key cho scalability
   - Khi nào cần chuyển từ DuckDB sang BigQuery/Snowflake production warehouse
   - Code example: incremental ingestr asset config với date filter

3. IDEMPOTENCY: Chạy pipeline 2 lần cùng date range → data có bị duplicate không
   - Ingestr asset: idempotent mặc định hay không?
   - SQL transform asset: view vs table — cái nào idempotent?
   - Incremental asset với unique_key: idempotent với merge strategy
   - Thế nào là "truly idempotent" Bruin pipeline
   - Code example: verify idempotency bằng row count check trước/sau rerun

4. DATA QUALITY: Làm sao biết data đúng sau khi pipeline chạy
   - Bruin built-in quality checks: loại nào có sẵn, config thế nào
   - Custom quality assertions: viết thế nào trong asset YAML block
   - Source freshness: detect khi source data stale
   - Row count anomaly detection: pipeline thành công nhưng data đột nhiên giảm 90%
   - Code example: quality check config trong asset YAML

5. COST & PERFORMANCE: Production pipeline — tránh reprocess không cần thiết
   - Full reload vs incremental: khi nào chọn cái nào
   - Date range parameterization: --start-date --end-date flags trong bruin run
   - Parallel execution: bruin run concurrency config
   - Asset caching: Bruin có cache intermediate results không
   - Code example: bruin run với date range flags tối ưu cost

6. SECURITY & GOVERNANCE: Những rủi ro trong setup hiện tại
   - Credentials trong connection config: không commit vào git — đúng pattern là gì
   - Environment separation: dev vs staging vs prod connection isolation
   - Bruin Cloud: credential management và access control
   - Bruin MCP: AI có access connection credentials không, rủi ro gì
   - Code example: environment-based connection config pattern
