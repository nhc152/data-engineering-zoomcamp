Nội dung bắt buộc:
- Stateless vs stateful processing.
- Event time vs processing time.
- Watermark là gì, vì sao cần cho window.
- Tumbling, sliding, session windows.
- Late events và allowed lateness.
- Checkpoint, savepoint, state backend.
- Exactly-once: điều kiện cần ở source + processor + sink.
- Flink architecture: JobManager, TaskManager, task slot, parallelism.
- PyFlink Table API vs DataStream API.
- JDBC sink upsert với primary key `NOT ENFORCED`.
- Mental model: `Window = bucket`, `Watermark = khi nào đóng bucket`, `Upsert = sửa kết quả khi event đến muộn`.
- 6 quiz:

  2. Event đến sau watermark thì chuyện gì xảy ra?
  3. Vì sao aggregation sink cần primary key?
  4. Checkpoint khác savepoint thế nào?

- Schema Registry: subject, compatibility mode, evolution.
- Kafka Streams: stream, table, joins, windows, state store.
- Spark Structured Streaming: readStream/writeStream, trigger, checkpoint, output mode.
- Faust/ksqlDB: high-level streaming abstractions.
- Outbox pattern, CDC, event sourcing, stream-table duality.
- Senior trade-off: operational cost của streaming vs business latency requirement.

  3. Kafka Streams state store dùng để làm gì?
  4. Spark Structured Streaming micro-batch khác Flink continuous thế nào?

### NỘI DUNG S4 - Production Readiness

Tạo 8 câu hỏi production. Mỗi câu phải có:
- Context thực tế.
- Risk.
- Cách debug.
- Cách fix.
- Code/config example.

1. SECURITY - Broker và database đang plaintext/hardcoded
   - Hardcoded `postgres/postgres`, `localhost:9092`, `redpanda:29092`.
   - No TLS/SASL.
   - Fix: env vars, Docker secrets, SASL_SSL, least privilege DB user.
   - Code example: compose env var + Python `os.environ`.

2. DELIVERY SEMANTICS - At-least-once vs exactly-once
   - Producer no idempotence.
   - Flink checkpointing local/non-durable.
   - JDBC sink upsert depends on primary key.
   - Fix: idempotent producer, transactions where possible, durable checkpoint storage, deterministic keys.

3. OFFSET & REPROCESSING
   - `earliest-offset` good for replay, risky for production restart if job identity changes.
   - Checkpoint vs new job submission.
   - Fix: savepoint strategy, deployment policy, runbook.

4. SCHEMA EVOLUTION
   - Workshop main uses JSON with no schema registry.
   - Avro extras show schema but not integrated into main flow.
   - Fix: Schema Registry, compatibility mode, contract tests.
   - Include example Avro compatibility decision.

5. LATE EVENTS & WATERMARK POLICY
   - Watermark 5 seconds is arbitrary.
   - Business latency vs completeness.
   - Fix: measure lateness distribution, configure allowed lateness/side output/upsert.

6. BACKPRESSURE & SCALING
   - Single Redpanda broker.
   - Flink parallelism 3, task slots 15.
   - Partition count not designed.
   - Fix: partition key, broker replication, consumer parallelism, Flink UI backpressure diagnostics.

7. OBSERVABILITY
   - Missing Kafka lag monitoring, Flink metrics, DB sink metrics, structured logs.
   - Fix: Prometheus/Grafana, Redpanda Console, Flink metrics reporter, alert thresholds.
   - Include logging schema: run_id, topic, partition, offset, event_time, processing_time, sink_status.

8. DATA QUALITY & DLQ
