1. CẤU TRÚC THƯ MỤC
═══════════════════════════════

═══════════════════════════════
2. NỘI DUNG CÁC FILE .MD
═══════════════════════════════
Tóm tắt từng file .md theo format:
[Tên file]
- Dạy về: [topic]
- Concepts chính: [list]
- Commands được dùng: [list commands thực tế]
- Lưu ý đặc biệt: [nếu có]

═══════════════════════════════
3. PHÂN TÍCH CODE FILES & CONFIGS
═══════════════════════════════
Với mỗi file config / code xuất hiện trong các .md hoặc trong repo, ghi rõ:

[pipeline config files - ví dụ: pipeline.yml, *.yaml, *.json]
- Assets được định nghĩa: tên, type, upstream/downstream dependencies
- Connections: tên, type, credentials
- Schedule / trigger settings
- Copy nguyên văn toàn bộ file

[asset files - ví dụ: *.sql, *.py dùng làm assets]
- Asset type (sql, python, ingestr...)
- Parameters / variables nhận vào
- Logic chính từng bước
- Upstream dependencies
- Copy nguyên văn toàn bộ file

[bruin CLI config - ví dụ: .bruin.yml, connections config]
- Environments được định nghĩa
- Connection types và fields
- Copy nguyên văn toàn bộ file

[pyproject.toml / requirements.txt / package.json nếu có]
- Dependencies list
- Copy nguyên văn toàn bộ file

[Bất kỳ script shell / Makefile / helper script nào khác]
- Làm gì, arguments, cách chạy
- Copy nguyên văn toàn bộ file

═══════════════════════════════
4. DATA FLOW THỰC TẾ
═══════════════════════════════
- Flow đầy đủ từng bước: data đi từ đâu → qua đâu → đến đâu
- Asset nào phụ thuộc asset nào (DAG dependency graph)
- File nào trigger file nào
- Thứ tự chạy đúng (materialization order)
- Dependency giữa các pipelines / assets

5. ĐIỂM YẾU & THIẾU SÓT
═══════════════════════════════

- Idempotency issues của các assets
- Production readiness gaps (error handling, retry, alerting...)
- Observability / monitoring gaps
- Những gì repo dạy nhưng không đủ cho Senior DE
