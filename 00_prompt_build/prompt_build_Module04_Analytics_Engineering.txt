- Giải thích được ELT vs ETL và tại sao dbt phù hợp với ELT
- Setup được local DuckDB hoặc BigQuery + dbt Cloud không cần hỏi
- Chạy được dbt deps, dbt seed, dbt run, dbt test không lỗi
- Đọc hiểu được lineage graph trong dbt docs
- Biết phân biệt ref() vs source() và dùng đúng trong model
- Biết dbt_project.yml cấu hình materialization mặc định thế nào

Intermediate (6 items):
- Tự viết staging model từ đầu: rename, cast, filter, macro call
- Tự viết schema.yml với tests: unique, not_null, accepted_values, relationships
- Hiểu incremental model: unique_key, merge strategy, --full-refresh khi nào
- Debug được lỗi ref() và source() không cần nhìn docs
- Dùng được dbt build --select với selector syntax để chạy subset
- Viết được macro đơn giản với Jinja và khai báo trong macros_properties.yml

Senior (6 items):
- Thiết kế được 3-layer architecture (staging/intermediate/marts) cho domain mới
- Implement được incremental model với lookback window tránh late-arriving data
- Viết được unit test cho macro và singular test cho business rule
- Cấu hình được slim CI workflow (state:modified+ + defer to prod)
- Giải thích được trade-offs materialization cho từng layer, justify cho người không biết dbt
- Trả lời được 4 câu Senior interview không cần nhìn tài liệu

Summary grid (5 điểm chốt):
1. dbt = version control + CI/CD + docs + lineage cho SQL transformation
2. 3-layer architecture (staging → intermediate → marts) = nguyên tắc nền tảng
3. Incremental + unique_key = idempotency cho fact tables lớn
4. dbt build + slim CI = gold standard cho production pipeline
5. AWS/Snowflake mapping — kiến thức transfer được:
   BigQuery → Redshift/Snowflake, DuckDB → SQLite/local dev,
   dbt Core/Cloud concept giống nhau 100% bất kể warehouse
