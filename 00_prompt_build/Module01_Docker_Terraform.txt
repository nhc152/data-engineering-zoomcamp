1. FAILURE: Pipeline ingest_data.py fail lúc 3h sáng sau khi insert được 500k/1M rows
   - Log xem ở đâu? (docker logs, PostgreSQL logs)
   - Data đang ở trạng thái gì? (partial insert)
   - Restart an toàn không? Tại sao không nếu không có idempotency
   - Fix ngay: query kiểm tra, truncate và chạy lại vs resume từ chỗ dừng

2. SCALE: Từ 1 file CSV 500MB lên 50 files tổng 50GB mỗi ngày
   - Bottleneck ở đâu? (single Python process, single PostgreSQL)
   - Docker single container không đủ → cần gì thay thế
   - Terraform: provision thêm resources tự động thế nào

3. IDEMPOTENCY: Chạy ingest_data.py 2 lần → duplicate data
   - Tại sao script hiện tại không idempotent
   - Fix: TRUNCATE trước INSERT vs ON CONFLICT DO UPDATE
   - Code example: thêm idempotency vào ingest_data.py

4. MONITORING: Làm sao biết pipeline chạy xong và đúng
   - Docker: health check config trong compose
   - PostgreSQL: query kiểm tra row count, max timestamp
   - Alert đơn giản: script gửi email/slack khi fail
   - Code example: healthcheck block trong docker-compose

5. COST: GCP resources này tốn bao nhiêu
   - GCS bucket: pricing model, estimate cho 50GB/tháng
   - BigQuery dataset: storage vs query cost
   - Terraform: tự động destroy dev resources ngoài giờ làm việc
   - Code example: scheduled terraform destroy

6. SECURITY: Những rủi ro bảo mật trong setup hiện tại
   - Credentials hardcode trong docker-compose → rủi ro gì
   - .env file có nên commit vào git không → .gitignore
   - Container đang chạy với root user theo mặc định — rủi ro gì?
  → Nếu có lỗ hổng trong app, attacker có full root access vào container
  → Fix: thêm vào Dockerfile:
    RUN addgroup --system appgroup && adduser --system --ingroup appgroup appuser
    USER appuser
  
- Service Account Key JSON trên máy cá nhân — rủi ro gì và alternative tốt hơn?
  → Key JSON nếu bị leak (commit nhầm lên git, máy bị hack) → attacker có full quyền trên GCP
  → Alternative production: Workload Identity Federation — GCP cấp quyền tạm thời
    cho workload (GitHub Actions, VM) mà không cần key file
  → Nguyên tắc: credentials không nên tồn tại dưới dạng file trên disk
   - Terraform state chứa sensitive data → remote state encryption
   - Code example: dùng Secret Manager thay vì biến môi trường
