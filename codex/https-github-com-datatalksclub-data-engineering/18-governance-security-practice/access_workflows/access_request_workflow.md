# Access Request Workflow

## Standard Access Request

Requester must provide:

- requester name/team
- dataset/table requested
- columns needed
- business justification
- duration
- whether export is needed
- ticket/link
- manager or data owner approval

## Approval Rules

| Data Classification | Required Approval |
|---|---|
| Public | none |
| Internal | role/group approval |
| Confidential | data owner approval |
| Restricted | data owner + security/privacy approval |

## Review Questions

- Can a curated/masked dataset satisfy the request?
- Are raw identifiers required?
- Can access be time-boxed?
- Should row-level filtering apply?
- Does requester need export permission?

## Access Expiration

- Internal: review quarterly.
- Confidential: review monthly/quarterly.
- Restricted: time-boxed by default, usually 7-30 days.
- Breakglass: expire within 24 hours unless extended.
