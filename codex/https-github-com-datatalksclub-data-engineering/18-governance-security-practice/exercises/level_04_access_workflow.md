# Level 04 - Access Workflow

## Goal

Practice approving or rejecting access requests.

## Scenario

A product analyst asks for raw customer table access to analyze retention.

## Tasks

1. Read `access_workflows/access_request_workflow.md`.
2. Decide if raw access is necessary.
3. Propose a curated alternative.
4. Define approval and expiration.
5. Write response to requester.

## Expected Decision

Default answer should not be raw access. Provide a masked/customer segment mart unless raw identifiers are truly required.
