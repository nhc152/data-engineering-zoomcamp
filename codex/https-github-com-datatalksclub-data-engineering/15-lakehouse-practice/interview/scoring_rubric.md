# Lakehouse Interview Scoring Rubric

## Strong Candidate

- Distinguishes Parquet file format from lakehouse table format.
- Explains metadata/snapshots/transaction logs.
- Compares Delta/Iceberg/Hudi by workload and ecosystem.
- Mentions compaction, vacuum, retention, catalog, schema evolution.
- Explains when not to use lakehouse.
- Talks about operational ownership.

## Medium Candidate

- Knows bronze/silver/gold.
- Knows Delta/Iceberg/Hudi names.
- Understands time travel and merge at high level.
- Weak on catalog and operations.

## Weak Candidate

- Says lakehouse means "store Parquet on S3".
- Cannot explain transaction log or snapshots.
- Cannot discuss failure modes.
- Says one format is always best.
- Ignores compaction/vacuum/catalog.
