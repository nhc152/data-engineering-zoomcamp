# Interview Readiness Checklist

## Project Explanation

- [ ] I can explain the business problem in 30 seconds.
- [ ] I can draw the architecture from memory.
- [ ] I can explain each tool choice.
- [ ] I can explain the data model and grain.
- [ ] I can explain one failure scenario.
- [ ] I can explain one cost/scaling trade-off.

## Debugging

- [ ] I can debug duplicate revenue.
- [ ] I can debug missing data.
- [ ] I can debug stale dashboard.
- [ ] I can debug late-arriving data.
- [ ] I can debug expensive query.

## Trade-offs

- [ ] Batch vs streaming.
- [ ] ETL vs ELT.
- [ ] Full refresh vs incremental.
- [ ] Warehouse SQL vs Spark.
- [ ] Managed service vs self-hosted.

## System Design

- [ ] I can estimate data volume.
- [ ] I can identify bottlenecks.
- [ ] I can design replay/backfill.
- [ ] I can define quality checks.
- [ ] I can discuss reliability and cost.
