# README Review Checklist

Use this before publishing a project.

## First 30 Seconds

- [ ] Can a reviewer understand the problem quickly?
- [ ] Is architecture visible near the top?
- [ ] Is the tech stack connected to reasons, not just listed?

## Reproducibility

- [ ] Setup commands are complete.
- [ ] Run commands are complete.
- [ ] Test commands are complete.
- [ ] Required credentials are explained safely.
- [ ] Expected outputs are shown.

## Data Engineering Quality

- [ ] Raw/staging/mart layers are explained.
- [ ] Data model and grain are documented.
- [ ] Data quality checks are documented.
- [ ] Incremental/backfill strategy is documented.
- [ ] Failure handling is documented.
- [ ] Cost/scaling notes exist.

## Clarity

- [ ] No vague phrases like "I used many tools."
- [ ] No unexplained architecture boxes.
- [ ] No missing assumptions.
- [ ] No hidden manual steps.
