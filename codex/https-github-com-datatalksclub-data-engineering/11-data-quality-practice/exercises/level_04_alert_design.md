# Level 04 - Alert Design

## Goal

Turn raw check failures into actionable alerts.

## Tasks

1. Read `broken/05_alert_without_owner.md`.
2. Rewrite the bad alert using `solutions/01_alert_templates.md`.
3. Add owner, impact, expected, actual, runbook.
4. Decide severity.

## Expected Learning

Alerts without owner and impact create noise. Good alerts route work to the right team.

## Deliverable

Create three alerts:

- freshness breach
- duplicate key
- new status value
