# Level 01 - Basic Quality Checks

## Goal

Write and interpret basic data quality checks.

## Tasks

1. Run freshness checks.
2. Run uniqueness checks.
3. Run accepted values checks.
4. Run not-null checks.
5. Document expected vs actual result.

## Expected Learning

- Passing job status is not enough.
- A check must be actionable.
- Checks should identify the failed table/key/date.

## Deliverable

Create notes:

```text
notes/level_01_quality_checks.md
```

Include:

- check name
- expected result
- actual result
- action if failed
