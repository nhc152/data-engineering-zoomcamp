# Level 05 - Incident and Postmortem

## Goal

Practice incident review and prevention.

## Tasks

1. Read all files in `incidents/`.
2. Pick one incident.
3. Fill `templates/postmortem_template.md`.
4. Add at least 3 prevention action items.
5. Assign owners and priority.

## Expected Output

Create:

```text
notes/postmortem_<incident_name>.md
```

## Evaluation

A strong postmortem:

- describes business impact
- gives timeline
- identifies root cause
- identifies detection gap
- creates concrete prevention actions
- avoids blame
